核心功能

多场景 AI 问诊

5 个垂直医疗 Agent（智能导诊 / 用药咨询 / 情绪疏导 / 营养指导 / 科室介绍），前端抽象统一 AIChat 组件 + useChatApi Hook，SSE 流式打字机效果。

components/AIChat + composables/useChatApi.js

BLE 智能拐杖

通过蓝牙 Notify 实时接收 S1/S2/S3 姿态数据。S3 持续 2 秒触发危险阈值时，自动通过 Android 原生 Intent 拨打紧急联系人。

stores/bluetooth.js + Bluetooth_crutch.vue

手语识别

Web-View 内独立运行摄像头采集。FPS_LADDER 11 档自适应帧率、运动检测预过滤，静止帧过滤率可达 40%，通过 Socket.IO 回传服务端。

Engineering Highlights

技术亮点

可讲的工程细节。代码开源可查。

01

Hybrid 架构：按需分层，不滥用 WebView

项目只在 渲染热路径（手语每帧 Canvas 像素操作）和 第三方 Web SDK（蜂鸟 fengmap 室内地图）两种场景用 Web-View。其余 UI 保留在 Vue 组件层以维持响应式。归档了 4 个旧版未使用 HTML。

02

自适应帧率 + 静止帧预过滤

sign.html 内置 FPS_LADDER 11 档阶梯（3/5/8/10/12/15/20/25/30/45/60）+ 激进降档 / 保守升档策略。运动检测让静止场景过滤率约 40%，减少服务端算力与带宽浪费。

03

Telemetry HUD：真数据验证

页面内嵌实时指标面板（FPS / 丢帧率 / 上行频率 / RTT）+ JSON/Markdown 导出。配合 headless Chrome 自动化脚本做回归测试。

04

可复用 AI 对话架构

useChatApi(agentId, apiBaseUrl) 工厂式 composable + 通用 AIChat 组件，新增一个医疗垂直场景仅需配置 agent 路由。Pinia 按 agentId 分桶持久化多场景上下文。

05

BLE 协议务实主义

厂家固件 ASCII 明文协议（S1:x#,S2:x#,S3:x#）带宽充裕、现场调试友好，评估后不要求修改。前端只做容错解析与持续 2 秒阈值防抖触发紧急呼叫。

06

集中化配置管理

config/env.js 统一出口所有后端地址（7 个不同端口的服务），18 个 Vue 页面从硬编码 URL 迁到常量引用。切换环境改一个文件即可。
